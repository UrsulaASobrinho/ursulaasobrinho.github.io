Dockerfile

top


existe um modelo BASE de comandos no dockerfile


FROM ubuntu:22.04
RUN apt-get update -y
RUN apt-get install apache2 -y
EXPOSE 80
CMD [“/usr/local/apache/bin/apachectl”, “-D”, “FOREGROUND”]


Entendendo

Linha Comandos dockerfile explicação
1 FROM ubuntu:22.04 FROM --> È A PRIMEIRA LINHA/COMANDO aqui Criar uma imagem base, ou imagem pai. A imagem pai vem de uma biblioteca de imagens no caso DcockerHub. ficando assim FROM(comando) ubuntu(nome da imagem) : 22.04 (sua versão)
2 RUN apt-get update -y RUN apt-get update -y --> RUN(comando de execuçao) apt-get update (Atualiza a imagem) , o -y é para não parar a execução com a pergunta confirma(sim ou não)
3 RUN apt-get install apache2 -y RUN apt-get install apache2 -y --> Aqui instala o servidor, no exemplo está apache2, poderia ser node.js, entre outros
4 EXPOSE 80 EXPOSE 80 --> disponibiliza a porta no containner
5 CMD [“/usr/local/apache/bin/apachectl”, “-D”, “FOREGROUND”] CMD [“/usr/local/apache/bin/apachectl”, “-D”, “FOREGROUND”] --> CMD é o comando do prompt ou seja vá par alinha de comando, os outros são para executar no LOCAL e suas permisoes , certamente muda de acordo com qual serviço esta instalando.



Pode acrescentar vários outros comandos no dockerfile, como criar um pasta MKDIR, ou copiar um arquivo por exemplo as variaveis de ambinte pre confic=guradas.
