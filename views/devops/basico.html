Git, GitHub e GitLab



O que e

O Git - um Sistema de Controle de Versoes. Tipo quando faço alteracoes em um sistema, etiquetamos aquela alteracao, jogando a informacao das alteracoes feitas, e quando essas alteracoes nao ficaram corretas, podemos retornar a outra versão antes da alteraco sem problemas.

O GitHub - é uma plataforma totalmente online e hospedar neles seus projetos, aqui onde o armazenamos as versoes de colaborar com softwares open source, seguir outros(as) programadores(as) e interagir com codigos de terceiros. de forma publica ou de forma particular.

O que os dois formam em comum

Bem simples no Git versionamos no GitHub armazenamos, porém de grande utilizade - o GitHub totalmente online - podemos compartilhar os sistemas com diversos coloboradores e todos interagir no mesmo com alteraçoes sem perda de versões Em diversas partes do sistemas por diversos programadores que compartilham um projeto. E evita conflitos de versoes tipo se alguem fez a mesma alteracao no mesmo lugar existe um controle de alerta.

O GitLab - digamos que seja uma versao mais utilizada para determinados necessidades de forma simples faz a mesma coisa que o GitHub porém para DevOps vai alem disso. Tipo, o versionamento criar ramificacoes. um processo de teste de varias etapas.

O que e Imagem

indice

Bem simples , no celular precisamos do Android ou OS para utilizar-lo. No computador preisamos de win ou linux (exemplo) para utilizar-mos. Então no docker precisamos da imagem.
Onde encontramos as imagens para Baixar ? No DockerHub existe a maior concentração de imagens e pode baixar com segurança.
Imagens oficiais e nao oficiais ? As imagens oficiais do docker sao aquelas sem usuários em seus nomes. A imagem “ubuntu:16.04″
Como criar imagens ? existem duas formas de criar imagens com commit e com Dockerfile.
Criando imagens com commit exemplo. Primeiro criamos um container qualquer:
docker container run -it --name containercriado ubuntu:16.04 bash
Criando imagens com Dockerfilet Neste opção se cria um arquivo com o nome docker file dentro os comandos de execucao.

Onde fica as imagens criados no docker/maquina local Uma vez baixado uma copia no seu computador ele fica armazenada '/var/lib/docker'
Precisamos de uma imagem para rodar um container Resumindo uma imagem e onde sera definido o que vai rodar ali dentro, (do container) quais as bibliotecas e configuracoes estarao presentes.

O que um container

indice

E um espaco isolado onde sera executado tudo que foi configurado na imagem. Imagina que seu computador tem wondows e nele esta configurado tudo que preciso para rodar um sistema em laravel e que tambem esta com node.js e tambem com Linux e nao conflitam entre si. Por que cada um esta ocupando espaço totalmente separado do outro. E que posso replicar esse mesmo senario em outra maquina e as limitacoes ou/e configuracoes de minha maquina local nao influenciara em nada.

O que precisamos saber sobre container
Pode iniciar um container sem nenhuma imagem base, mas nao tera nenhuma biblioteca ou comando para executar ali dentro.
Nao existe uma dependencia obrigatoria, mas geralmente se cria um novo container com base em uma imagem.
Os containers podem ser parados, reiniciados ou removidos.
Quando excluir o container tudo que existia dentro dele também sera excluido. Geralmente quando se quer guaradar cria um volume.

O que e volume


Uma forma de garantir a persistência de dados ao trabalhar em conteineres. independentemente do ciclo de vida do contêiner. Isso permite que os usuários façam backup de dados e compartilhem sistemas de arquivos entre conteineres com facilidade. indice

Comandos basicos


Existe um modelo BASE de comandos

Seguinte existe um repositorio/bibliotecas que disponibiliza imagens ja prontas e ate com configuracoes para certo ambientes, como Apache, Node.js, Wordpress entre outros. Para adquirir essa imagem, devemos uscar atravez de alguns comandos. Que uma fez executado baixa/uploads dentro do container.
No exemplo abaixo, estou montando um servidor Apache2
FROM ubuntu:22.04
RUN apt-get update -y
RUN apt-get install apache2 -y
EXPOSE 80
CMD ['/usr/local/apache/bin/apachectl', '-D', 'FOREGROUND']


Entendendo cada linha de comando

Linha Comandos dockerfile explicacao
1 FROM ubuntu:22.04 FROM --> E A PRIMEIRA LINHA/COMANDO aqui Criar uma imagem base, ou imagem pai. A imagem pai vem de uma biblioteca de imagens no caso DcockerHub. ficando assim FROM(comando) ubuntu(nome da imagem) : 22.04 (sua versao)
2 RUN apt-get update -y RUN apt-get update -y --> RUN(comando de execuçao) apt-get update (Atualiza a imagem) , o -y e para não parar a execucao com a pergunta confirma(sim ou nao)
3 RUN apt-get install apache2 -y RUN apt-get install apache2 -y --> Aqui instala o servidor, no exemplo está apache2, poderia ser node.js, entre outros
4 EXPOSE 80 EXPOSE 80 --> disponibiliza a porta no containner
5 CMD ['/usr/local/apache/bin/apachectl', '-D', “FOREGROUND”] CMD ['/usr/local/apache/bin/apachectl', '-D', “FOREGROUND”] --> CMD e o comando do prompt ou seja va par alinha de comando, os outros sao para executar no LOCAL e suas permisoes , certamente muda de acordo com qual servico esta instalando.

Pode acrescentar varios outros comandos no dockerfile, como criar um pasta MKDIR, ou copiar um arquivo por exemplo as variaveis de ambinte pre configuradas.

Dockerfile

Para que serve - um arquivo que junta os comandos de instalacao de uma imagem, podendo esta imagem ser uma simple copia ou uma imagem configurada com parametrizacao/necessidades do aplicatico.
indice

Criando um arquivo dockerfile


Partirndo do principio que ja tenha instalado no seu pc docker, visual Studio Code, e de preferecia tambem o git

mkdir website [enter] Passos simples, crie um diretorio
cd website [enter]
code . [enter] uam vez dentro do diretorio criado chame o visual studio code
Uma vez dentro no vscode crie o arquivos dockerfile, veja esse exemplo abaixo:

# dockerfile explica linha a linha
1 FROM ubuntu Comando que inicia a execucao ja sabemos que e a primeira linha
2 RUN apt-get update -y atualzia
3 RUN apt-get install apache2 -y instala no caso aqui a opcao foi servdor apache
4 # Variaveis ​​de ambiente no Apache, estas variaveis estao vinculadas apache2.conf
4 ENV APACHE_LOCK_DIR="/var/lock/apache2" aqui ja sao aqueles comoando extra, neste caso estou configurando as variaveis de ambiente do apache
5 ENV APACHE_PID_FILE="/var/run/apache2/apache2.pid"
6 ENV APACHE_RUN_USER="www-data"
7 ENV APACHE_RUN_GROUP="www-data"
8 ENV APACHE_LOG_DIR="/var/log/apache2"
9 ENV APACHE_SERVER_NAME="localhost"
10 VOLUME /var/www/html/ Aqui monta o volume que sera o diretorio onde vamos colocar o projeto/app
11 #Este e o arquivo de configuracao principal do servidor.
12 COPY apache-conf /etc/apache2/apache2.conf Aqui quando criei o diretorio no pc copiei para a pasta esse arquivo, que faz as varieaveis de ambiente que configurei acima
13 #Expose Apache
14 EXPOSE 80 Aqui determina qual porta sera disponibilizada no container
15 # Launch Apache2
16 CMD ['/usr/sbin/apache2ctl', '-DFOREGROUND'] Sempre a ultima linha executa observe que o caminho e configuracao do apace

Executando o dockerfile


NO Vcode va para um terminal
vamos gerar a imagem, com o seguinyte comando
# comando explica linha a linha
1 docker image build -t website . docker image build -t nome da imagem (ponto) qoer dizer neste diretorio. Automaticamente va procurar o dockerfile
2 docker image ls Este comando mostra a imagem criada
3 docker container run -d -p8081:80 website cria o conatiner (onde vamos colocar o projeto) run(executa) -d -p(porta do PC local):(porta que se colocou no dockerfile) nome da imagem que gerou e vai utilizar

Agora e so testar abrindo um browser e digitar http://localhost:8081/

No terminal que abriu dentro do vscode, tem quer saber qual e o nome do container_id
